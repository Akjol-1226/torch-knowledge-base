# T-009 — core/db SQLModel engine + 测试 fixture

## 改了什么
- 新增 `backend/app/core/db.py`（async engine + sessionmaker + init_db + get_db 依赖）
- 新增 `backend/tests/conftest.py`（db_engine + db_session fixture，跨 unit/integration 复用）
- 新增 `backend/tests/unit/test_db.py`（1 个 round-trip 单测）
- 改 `backend/app/main.py` lifespan 里加 `await init_db()`
- 改 `backend/pyproject.toml` 补 `greenlet>=3.0.0` 依赖

## 为什么改
所有模块的数据访问都基于 `AsyncSession`，统一在 `core/db` 抽 engine/session factory，业务模块通过 `Depends(get_db)` 注入。

## 思路
- engine 和 session factory 用模块级 lazy 单例（首次 get_engine 时读 settings 创建）
- 测试用 in-memory SQLite（`sqlite+aiosqlite:///:memory:`），每个 test function 独立 engine，避免污染
- conftest 在 `tests/` 顶层，让 unit + integration 都能用 `db_session` fixture

## 验收
- `uv run pytest tests/unit/test_db.py -v` → 1 passed
- 也跑 health test 确认 main.py 改动没破坏：tests/integration/test_health.py 仍 2 passed

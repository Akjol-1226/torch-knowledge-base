# T-007 — core/config + core/logging

## 改了什么
- 新增 `backend/app/core/config.py`（pydantic-settings 加载 .env / 环境变量）
- 新增 `backend/app/core/logging.py`（structlog 配置 + dev/prod 渲染器切换）
- 新增 `backend/tests/unit/test_config.py`（2 个单测）

## 为什么改
基座第一块：所有业务模块都依赖 `Settings` 和 `get_logger`。先把配置和日志的契约定好。

## 思路
- Settings：LITELLM 三项必填（无默认值，强制 env 注入）；app_env / log_level / database_url 有默认值
- `get_settings()` 用 `@lru_cache` 单例化，避免每次 Depends 时重读 .env
- logging：dev 用 ConsoleRenderer（彩色对开发者友好），其他环境 JSONRenderer（机器可解析）

## 验收
- `uv run pytest tests/unit/test_config.py -v` → 2 passed
- test 用 `Settings(_env_file=None)` 避免读真实 .env 文件污染（plan 原样代码会因为没有 .env 默认值而 fail，这里改为 monkeypatch + 显式禁用 env_file）

# T-011 — core/vlm/client（LiteLLM 调用层）

## 改了什么
- 新增 `backend/app/core/vlm/client.py`（`VLMClient` 用 `AsyncOpenAI` 调 LiteLLM Proxy）
- 新增 `backend/app/core/vlm/smoke_test.py`（手动 smoke test 脚本，需 .env 真配齐）
- 新增 `backend/tests/unit/test_vlm_client.py`（2 个 mock 测试）

## 为什么改
统一 LLM 调用边界（AGENTS.md R-4），所有业务模块通过 `VLMClient.extract_page()` 调 VLM，不允许直连厂商 SDK。

## 思路
- LiteLLM 接入方式：HTTP Proxy + OpenAI 兼容协议（OQ-2 默认答案），用 `openai` Python SDK 配 `base_url`
- 模型名走 config 注入（`settings.litellm_default_vlm_model`），支持 per-call override（`model=` 参数）
- 输出统一 `VLMResponse` schema，含 raw_text / notsure_segments / model_id / 计费字段 / latency_ms
- smoke_test 用 1x1 透明 PNG 作 fallback fixture，避免依赖外部图片

## 验收
- `uv run pytest tests/unit/test_vlm_client.py -v` → 2 passed
- 真实 smoke test 没跑（按任务约束：无真实 LITELLM_BASE_URL/API_KEY）；脚本本身可 import 和启动，调用时会被 401/连接错误挡住，这是预期
